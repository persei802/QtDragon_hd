Z LEVEL COMP UTILITY

QtDragon can compensate for Z level height changes using it's own gcode generation code for probing. An external program like G-code Ripper is not required, but can be used if so desired.

Z level compensation is a bed levelling/distortion correction function typically used in 3D printing or engraving. It uses a HAL non-realtime component which utilizes the external offsets feature of LinuxCNC.

Whenever a gcode program is loaded:

  • zlevel determines the currently loaded program file and looks for a related probe result file
  • for example, if some_file.ngc is loaded, it looks for a file called probe_some_file.txt
  • if it exists, the data is read and sent to the compensation module via a shared memory structure
  • a surface map is created and displayed in the zlevel module
  • if the COMP ENABLE button is checked, the compensation module sends offset data to the Z axis

Creating a Probe File

  • from the FILE page, load the gcode program which needs to be level compensated
  • the X and Y dimensions are automatically calculated and shown in the parameter frame in the X and Y size fields
  • enter the number of desired probe points for each axis
  • ensure that the selected zero reference point is the same as that of the gcode program
  • before a program is saved or sent to linuxcnc, the input parameters are checked for errors
  • any errors that are found are highlighted with a red border

The input data parameters are checked and if found valid, a dialog box prompts for a save filename. The probe program is calculated and saved to the specified filename. If the returned save file name doesn't start with 'probe_', it is added. When the file is loaded and run, it will produce a file with the same name but ending with .txt. This is the compensation data file and contains the probed values.

The input parameters are checked and if found valid, a temporary file is created and loaded to Linuxcnc. If this file is run, it will produce a file in the users CONFIG directory called probe_temp.txt

Sample postgui HAL file for combined spindle raise and Z Level compensation

# load a summing component for adding spindle lift and Z compensation
loadrt scaled_s32_sums
addf scaled-s32-sums.0 servo-thread
 
loadusr -Wn compensate python3 lib/compensate.py
 
# Z level compensation
net compensate-on    compensate.enable <= zlevel.enable
setp compensate.scale 1000
 
# add Z level and scaled spindle raise level values together
net eoffset-count    scaled-s32-sums.0.in0   <= qtdragon.eoffset-count
net comp-count       scaled-s32-sums.0.in1   => compensate.counts
net zaxis-eoffset    scaled-s32-sums.0.out-s => axis.z.eoffset-counts
setp scaled-s32-sums.0.scale0 1000
setp axis.z.eoffset-enable True
setp axis.z.eoffset-scale 0.001
